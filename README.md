# Чат с ИИ

Веб-приложение для общения с ИИ: текстовый и голосовой ввод, ответы через OpenAI API (или демо-режим без ключа).

---

## Как это устроено

- **Клиент** (React + Vite + Tailwind) — форма с полем ввода, кнопкой голосового ввода и областью ответа. Отправляет сообщения на бэкенд по `POST /api/chat`.
- **Сервер** (Express) — принимает запросы, валидирует тело, вызывает сервис чата и отдаёт ответ в формате `{ reply: "..." }`.
- **Сервис чата** (`server/src/services/chat.service.js`) — решает, вызвать ли реальный OpenAI API или вернуть **fallback-ответ** (демо-режим). Ответ в любом случае возвращается в том же формате, поэтому фронтенд не знает, был ли это ChatGPT или fallback.

Последовательность: пользователь вводит текст (или голос) → клиент шлёт `message` на сервер → сервис либо дергает OpenAI, либо формирует демо-ответ → клиент показывает `reply` в блоке ответа.

---

## Почему мы не всегда используем ChatGPT, а его fallback

Реальный ChatGPT (OpenAI API) используется только когда есть валидный API-ключ и запрос до него доходит. В остальных случаях приложение сознательно использует **fallback** — заранее сформированный демо-ответ, чтобы:

1. **Работать без платной квоты** — можно запустить и проверить весь сценарий (фронт ↔ бэк ↔ «ИИ») без ключа OpenAI и без трат.
2. **Не ломать UX при проблемах с ключом** — если ключа нет или он неверный, при включённом fallback пользователь видит осмысленное сообщение («демо-режим», причина), а не сырую ошибку.
3. **Мягко обрабатывать лимиты API** — при 401 (неверный ключ) или 429 (превышен лимит) сервер может вернуть тот же демо-ответ вместо ошибки, чтобы приложение оставалось рабочим в демо/тестовом режиме.

То есть fallback — это не «запасной вариант на случай сбоя», а **основной способ запустить и показать приложение без OpenAI** и при этом сохранить единый поток «вопрос → ответ» на фронте.

## Когда идёт запрос в ChatGPT, а когда — fallback

| Условие | Результат |
|--------|-----------|
| `AI_FORCE_MOCK=true` | Всегда fallback (демо-ответ), API не вызывается. |
| Нет `OPENAI_API_KEY` или ключ неверного формата | Fallback, если включён `AI_FALLBACK_ON_ERROR`; иначе ошибка 500. |
| Запрос к OpenAI вернул 401 или 429 | Fallback, если включён `AI_FALLBACK_ON_ERROR`; иначе ошибка 401/429. |
| Запрос к OpenAI успешен | Ответ ChatGPT (модель из `OPENAI_MODEL`, по умолчанию `gpt-4o-mini`). |
| Другая ошибка API (сеть, 5xx) | Ошибка 502 на клиенте, fallback не используется. |

Переменные окружения (в `server/.env` или в корне):

- `OPENAI_API_KEY` — ключ OpenAI (если не задан или неверный, при включённом fallback будет демо-режим).
- `AI_FORCE_MOCK=true` — всегда использовать только fallback, не вызывать API.
- `AI_FALLBACK_ON_ERROR=false` — отключить fallback при ошибках; при неверном ключе или 401/429 отдавать ошибку пользователю.

## Запуск

1. **Сервер** (из папки `server`):
   ```bash
   cd server
   npm install
   cp .env.example .env   # при необходимости, задать переменные
   npm run dev
   ```
   По умолчанию слушает порт 4000.

2. **Клиент** (из папки `client`):
   ```bash
   cd client
   npm install
   npm run dev
   ```
   Открыть в браузере указанный адрес (обычно `http://localhost:5173`). API по умолчанию берётся с `http://localhost:4000`; при необходимости задать `VITE_API_URL` в `client/.env`.

Для работы без OpenAI достаточно включить fallback (например, `AI_FORCE_MOCK=true` или не задавать ключ и оставить `AI_FALLBACK_ON_ERROR` по умолчанию).
